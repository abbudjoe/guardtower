# Daily Guardtower Automation Prompt

Use this prompt for the Codex cron automation that runs Guardtower. The full versioned automation definition lives next to this file in `daily-guardtower.automation.toml`.

```text
Run the Guardtower scanner with `python3 /Users/joseph/guard/plugins/guardtower/scripts/guardtower.py --config /Users/joseph/guard/plugins/guardtower/config.json`. The config's `env_file` is `/Users/joseph/guard/.env`, and the scanner loads credentials from that file itself. Do not inspect or print secret values. Do not claim `X_BEARER_TOKEN`, `VERCEL_TOKEN`, or other credentials are unset merely because they are absent from the worker shell; only report a token as missing/skipped if the scanner output, Markdown report, or JSON report explicitly records that source as skipped or failed. Then read the latest Markdown and JSON report in `/Users/joseph/.codex/guardtower/reports/` and summarize the Delta section first: new exposures, resolved exposures, still-present exposures, and exposures not observed because a source failed. Next summarize the Deployment Inventory and any deployment discovery failures, especially Vercel projects marked deployed, deployable, unknown, or unverified. Treat Vercel API evidence as deployment proof; do not treat local `vercel.json` markers alone as deployed. Then summarize the Remediation Plan: list the highest urgency package clusters with package, exposure count, affected directories, deployment status, severity class, recommended action, and attribution command. Then summarize the Action View only for any critical/deployed rows not already covered by the clusters. Ignore generic technology or AI-news roundup items unless the report ties them to a concrete security exploit, CVE, compromised package, or actively exploited vulnerability. After that, summarize source failures or skipped sources exactly as recorded by the report. Do not mutate cloud jobs or paid compute. Do not install packages.
```

Keep this template aligned with the saved `daily-guardtower` automation in Codex. After changing this file or `daily-guardtower.automation.toml`, update the live automation and run the Guardtower tests.
